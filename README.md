# PADII ★ RUL Engineer

Интерактивная **pixel-art демонстрация ML-модели прогнозирования RUL**
(Remaining Useful Life — остаточного ресурса авиадвигателей) в реальной
бизнес-задаче. Вы — главный инженер вымышленной авиакомпании **PADII**.

Игра показывает не метрику модели, а её **ценность в деньгах**:
насколько прогноз RUL влияет на инженерные решения, прибыль и риск аварий.

> **Принципиально:** модель **не обучается внутри игры**. Используются только
> заранее посчитанные прогнозы (precomputed predictions). Каждый клик
> «Continue Flight» обновляет состояние **мгновенно** — никакого ожидания.

---

## Что демонстрируется

| Решение | Эффект |
|---|---|
| Успешный рейс | **+$5,000** |
| Отправка на ТО | **−$50,000** |
| Авария в воздухе | **−$5,000,000** |

- **Польза модели** — прогноз RUL подсказывает, когда безопасно остановиться.
- **Цена ошибки** — если модель переоценила ресурс, борт может разрушиться в полёте.
- **Безопасность vs прибыль** — раннее ТО теряет выручку, позднее грозит аварией.
- **Сравнение моделей** — `Baseline` (точная, но рискует переоценить RUL) против
  `Safety-Aware` (осознанно занижает RUL → меньше аварий ценой части выручки).

## Данные и модель

Источник — реальный датасет **NASA C-MAPSS (FD001)**, прогон двигателей
до отказа. В игре используются **4 сенсора**:

| Сенсор | Колонка | Физический смысл |
|---|---|---|
| sensor 4 | 9 | Температура на выходе из турбины низкого давления |
| sensor 9 | 14 | Физическая скорость вращения ядра |
| sensor 11 | 16 | Статическое давление на выходе из компрессора ВД |
| sensor 14 | 19 | Скорректированная скорость вращения ядра |

`scripts/precompute.py` обучает модель **офлайн** по логике ноутбуков Week 6:
агрегированные признаки по operation settings + всем 21 сенсорам, RandomForest,
out-of-fold residual correction и safety-aware вариант. В игре при этом
отображаются только 4 основных сенсора.

- **baseline** — Lev Week 6 OOF residual correction;
- **safety_aware** — более осторожный вариант, снижающий риск переоценки RUL.

Игровые борта — holdout-двигатели из `train_FD001`: они исключаются из обучения,
но имеют полную траекторию до отказа, поэтому игра честно знает момент failure.

Результаты кладутся в `data/` и читаются игрой только на чтение.

## Запуск

```bash
# 1. зависимости
pip install -r requirements.txt

# 2. (опционально) перегенерировать прогнозы из CMAPSS
#    путь к CMAPSSData задаётся в scripts/precompute.py (CMAPSS_DIR)
python scripts/precompute.py

# 3. запустить игру
python -m streamlit run app.py
#    -> http://localhost:8501
```

Данные в `data/` уже сгенерированы — шаг 2 нужен только для пересборки.

## Деплой

Текущая версия — **Streamlit/Python-приложение**, поэтому её нельзя напрямую
задеплоить на GitHub Pages как обычный статический сайт. GitHub Pages отдаёт
только HTML/CSS/JS и не запускает Python-сервер.

Рекомендуемый вариант:

1. Залить репозиторий на GitHub вместе с `app.py`, `requirements.txt`, `game/`,
   `assets/` и уже сгенерированными `data/`.
2. Деплоить через Streamlit Community Cloud, Render или Railway.
3. В деплое указать main file: `app.py`.

Обучение модели не происходит на сервере при каждом запуске игры. Если нужно
обновить модель, запустите локально:

```bash
python scripts/precompute.py
```

затем закоммитьте обновлённые файлы из `data/`. Сервер будет только читать эти
precomputed predictions.

## Как играть

1. **Lobby** — выберите модель (`Baseline` / `Safety-Aware`) → **Start Campaign**.
2. **Airport** — кликните борт на карте или в правом списке Fleet.
3. **Console** — смотрите Predicted RUL, статус (Safe / Warning / Critical),
   график и сенсоры (клик по сенсору → его график, **Back** → назад).
4. **Решение** — `Continue Flight` (рейс, +$5k, +1 цикл) или
   `Send to Maintenance` (−$50k, эпизод завершён).
5. **Report** — после ТО/аварии раскрывается **True RUL** (пунктир на графике)
   и финансовый итог с объяснением.
6. **Campaign Summary** — итоговый бюджет, рейсы, ТО, аварии, средняя ошибка
   модели и бизнес-выводы.

### Статусы (по предсказанному RUL)
- 🟢 **Safe** — RUL > 20
- 🟡 **Warning** — 5 ≤ RUL ≤ 20
- 🔴 **Critical** — RUL < 5

## Расширение

- Новые борта/двигатели — добавьте строки в `data/*` (формат tidy расширяемый).
- Другой датасет CMAPSS (FD002–FD004) — поменяйте файл в `precompute.py`.
- Новые сенсоры — добавьте в `GAME_SENSORS` и `sensor_catalog.json`.
